# 架构总览

## 一句话定义

这是一套面向智能体的分层记忆架构：把短摘要、结构化决策、时间线事件、原话证据分开存储，再通过召回、重排和上下文打包，把最相关的记忆注入当前运行时上下文。

## 高层流程

```text
查询
  -> 意图分类
  -> 多源召回
  -> 结果合并
  -> 重排
  -> 上下文打包
  -> 运行时注入
```

## 召回来源

### 1）L0
用于快速加载上下文的短摘要记忆。

### 2）决策
用于保存可复用的结构化结论。

### 3）事件
用于表达任务推进过程和时间线。

### 4）回放
用于保存和回放原始对话证据。

### 5）混合检索
使用向量 + 词法检索提升召回质量。

## 设计目标

- 避免启动时硬读大记忆文件
- 改成按需查询记忆
- 不把所有记忆扁平化塞进同一个存储
- 在需要时保留回看原始措辞的能力
- 未来可以接入更大的智能体运行系统

## 一个关键设计选择

这个项目**不把 `event` 当成 L0 的替代品**。

后来做过一次重要修正：
- 增加独立的 L0 存储
- ingest 时显式生成 `l0Item`
- 把 `l0Items` 作为一等召回来源

原因是：
- 事件层表达的是过程推进
- L0 表达的是短摘要导航
- 两者职责不同，不能混用

## 当前状态

这套架构已经能在本地实验环境里运行，而当前仓库是它整理出来的独立发布版项目结构。
