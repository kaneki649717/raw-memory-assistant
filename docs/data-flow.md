# 数据流

## 写入路径

```text
用户回合 + 助手回合
  -> 实体提取
  -> 摘要生成
  -> 创建 event
  -> 按需创建 decision
  -> 创建 / 更新 L0 项
  -> 保存 replay 项
  -> 更新向量索引
```

## 读取路径

```text
查询
  -> 意图分类
  -> 从 L0 / decision / event / replay / hybrid 中召回
  -> 合并结果包
  -> 重排
  -> 构建 context pack
  -> 注入运行时上下文
```

## 为什么这样设计

这套架构不是把所有内容扔进一个统一检索空间里，而是按“用途”拆开记忆。

因此它可以更好地回答不同问题：

- **连续性问题**：优先决策和时间线
- **细节问题**：优先决策和片段证据
- **原话问题**：优先原话回放证据
- **快速启动问题**：优先 L0 短摘要
